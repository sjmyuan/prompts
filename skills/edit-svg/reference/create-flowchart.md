# Create Flowchart — Detailed Steps

Applies **create-flowchart** in the edit-svg skill.

Before starting, load the following from [reference/computation-snippets.md](computation-snippets.md):
- Script calling guide and snippet patterns — **Read the zero-tolerance rule at the top.**
- Also load [reference/design-standards.md](design-standards.md) for PPT-quality standards.

**Steps**:

0. **🔴 CRITICAL — No manual coordinate math**. Every position, dimension, path string, and SVG element MUST come from a script execution. Never add/subtract coordinates, verify alignment, or compute offsets in your reasoning. **Trust the script output.**

1. **Analyze the process**: Identify start/end nodes, process steps, decisions, and branching paths. Determine flow direction (default: top-to-bottom). Extract a diagram title.

2. **Build node/edge data**:
   - `nodes[]`: each with `id`, `type` (`start`/`end`/`process`/`decision`/`subprocess`/`document`/`data`/`connector`/`fork`), `text`, `row`, `col`.
   - `edges[]`: each with `id`, `from`, `to`, optional `label` and `branch` (`"yes"`/`"no"` for decision branches)

3. **Compute node positions via script**: Run the **Compute node positions** snippet from [reference/computation-snippets.md](computation-snippets.md) with your node list. **Do NOT compute any positions manually.**

4. **Route connections via script**: For each edge, run the **Route connection** snippet using source/destination node bounding boxes. **Do NOT construct any path `d` strings manually.**

5. **Generate SVG elements via script**: Run the **Generate SVG elements** snippet for each node and edge. **Do NOT write raw `<rect>`, `<path>`, or `<text>` elements manually.**

6. **Generate defs and decorations via script**: Call `colors.get_shadow_filter()` + `colors.get_gradient_defs()` + `svg_builder.generate_arrow_marker()` for `<defs>`. Call `svg_builder.generate_title_bar(title, width)` for the title bar. **Do NOT write defs or title bar manually.**

7. **Validate via script**: Run overlap and contrast validation snippets from [reference/computation-snippets.md](computation-snippets.md). Fix issues by adjusting row/col and re-running scripts — **do not fix by manually tweaking coordinates**.

8. **Compute viewBox via script**: Run the **Compute viewBox** snippet with all element bounding boxes.

9. **Assemble SVG**: Follow the **SVG assembly pattern** in [reference/computation-snippets.md](computation-snippets.md). All SVG fragments must be pre-generated by scripts.

10. **Final output**: Return raw, valid SVG code with no surrounding explanation.
