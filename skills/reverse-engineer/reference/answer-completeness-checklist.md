# Answer Completeness Checklist

Before presenting investigation findings to the user, verify all items below.

## Question Coverage
- [ ] The direct answer to the question is stated first and is concise
- [ ] All sub-questions or aspects of the original question are addressed
- [ ] If the investigation was incomplete, what was found and what remains unclear is explicitly stated

## Code References
- [ ] Key files are referenced with paths and line numbers
- [ ] Relevant code snippets are included for critical functions or classes
- [ ] References are accurate — paths and line numbers match the current codebase

## Implementation Detail
- [ ] Control flow is traced from entry point to exit
- [ ] Data flow is traced from source through transformations to output
- [ ] Dependencies (internal modules, external libraries, services) are listed
- [ ] Key design decisions, trade-offs, and patterns are noted

## Context & Implications
- [ ] How the answer fits into the broader system is explained
- [ ] Related components or patterns are mentioned
- [ ] Edge cases, limitations, or potential issues are identified

## Next Steps
- [ ] Related questions or deeper investigation paths are suggested
- [ ] Areas where the codebase is ambiguous or under-documented are flagged
